# Browser Patch Maintenance

Detailed browser patch upgrade notes are maintained outside the public
repository.

The public repository documents the build entry points and release checks needed
to work with VulpineOS at the runtime and orchestration layer. Low-level browser
patch migration steps, reject triage notes, and implementation-specific patch
playbooks should stay in private maintainer documentation.

Public contributors proposing browser-layer changes should open an issue first
so maintainers can confirm scope and run the appropriate private validation
gates.
